🔄 LangGraph vs LangChain

Característica LangChain LangGraph ✓
Cadenas lineales ✓ ✓
Decisiones dinámicas ✗ ✓
Ciclos/Reintento ✗ ✓
Reflection Pattern ✗ ✓
HITL (Human Approval) ✗ ✓

Conclusión: LangGraph permite decisiones dinámicas = esencial para patrones avanzados

🛠️ 3 Tools Nativos

1. get_influence_metrics()

Identifica usuarios influyentes basado en engagement y reach

Retorna: JSON con ranking de autores y scores

2. analyze_sentiment()

Calcula distribución de sentimiento (POSITIVE, NEGATIVE, NEUTRAL, UNKNOWN)

Retorna: Porcentajes por categoría

3. trace_propagation()

Rastrea cómo se propaga un post (árbol de retweets con BFS)

Retorna: Estructura JSON del árbol de propagación

🎯 Patrón 1: DEFAULT (Normal)

Usuario: "¿Quiénes son los influyentes?" ↓ Agent procesa → decide "necesito get_influence_metrics" ↓ Execute tool (HTTP request) ↓ Generate response ↓ Respuesta natural al usuario

✓ Rápido • ✓ Eficiente • Sin explicación del razonamiento

🧠 Patrón 2: ReAct (Reasoning)

ReAct = Thought → Action → Observation → Reflection

Thought: "Necesito identificar usuarios influyentes" ↓ Action: Ejecutar get_influence_metrics() ↓ Observation: Veo los resultados del tool ↓ Reflection: "La respuesta es clara y completa" ↓ Final Answer: Generamos respuesta natural

✓ Visible el razonamiento • Más lento (2 LLM calls)

🔍 Patrón 3: Reflection (Auto-evaluación)

¿Mi respuesta es suficiente? Si NO → reintento

Execute tool 1 ↓ [Reflection] ¿Es suficiente? ├─ INSUFFICIENT → reintenta (máx 1) └─ SUFFICIENT → continuar ↓ Execute tool 2 (si fue necesario) ↓ Generate response

✓ Auto-corrección • ✓ Mejor calidad • Más lento (evaluación LLM)

📋 Patrón 4: Planning (Descomposición)

Dividir pregunta compleja en pasos antes de ejecutar

Usuario: "Influencers + sentimiento?" ↓ [Planning] Generar plan: Paso 1: get_influence_metrics() Paso 2: analyze_sentiment() Paso 3: Combinar resultados ↓ Ejecutar paso 1 ✓ Ejecutar paso 2 ✓ Ejecutar paso 3 ✓ ↓ Respuesta estructurada

✓ Para queries complejas • ✓ Organizado

👤 Patrón 5: HITL (Human-in-the-Loop)

Pedir aprobación humana antes de ejecutar tools

Agent decide: "necesito ejecutar get_influence_metrics" ↓ [PAUSA] Mostrar al usuario: Tool: get_influence_metrics Input: {} ↓ Usuario responde: ¿si o no? ├─ SI → Ejecutar tool + responder └─ NO → Cancelar acción

✓ Control humano • ✓ IA segura • Requiere interacción manual

⚡ Flujo Condicional (Conditional Edges)

LangGraph decide en tiempo de ejecución qué nodo visitar 

def route_after_tool_selection(state):
    mode = state.get("pattern_mode")
    has_tool = state.get("last_tool_result", {}).get("is_tool_call")

    if not has_tool:
        return "generate_response"  # Sin tool
    if mode == "react":
        return "node_react_think"   # Razonamiento
    if mode == "hitl":
        return "node_hitl_check"    # Aprobación
    if mode == "planning":
        return "node_plan"          # Descomposición
    return "execute_tool"           # Normal

✓ Mismo agente, 5 comportamientos diferentes

🔒 Seguridad 1: Detección de Inyecciones

Detectar ataques: "Olvida instrucciones anteriores"

30+ patrones regex para detectar:

  • ignore, forget, bypass
  • system prompt, instructions
  • jailbreak, exploit
  • Variaciones en español e inglés

Severidad: LOW • MEDIUM • HIGH

Rate Limiting: Máximo 20 requests/minuto por usuario

🔐 Seguridad 2: Detección y Masking de PII

Detectar y ocultar datos personales sensibles

Tipos detectados:

  • Email → [MASKED-EMAIL]
  • Teléfono → [MASKED-PHONE]
  • Tarjeta crédito → [MASKED-CC]
  • SSN, Pasaporte, IP → Masking automático

Resultado: Usuario nunca ve datos sensibles
